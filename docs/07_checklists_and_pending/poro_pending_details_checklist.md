# 포로 서버 세부사항 추가 확정 필요 목록 초안

## 문서 목적
이 문서는 지금까지 큰 틀과 중간 설계를 많이 진행한 뒤,
아직 **최종 확정이 필요한 세부사항**을 한 번에 볼 수 있도록 정리한 체크리스트다.

핵심 목표:
- 무엇이 이미 정해졌고
- 무엇이 아직 권장안 수준이며
- 무엇을 나중에 테스트로 확정해야 하는지
구분하는 것이다.

---

## 1. 이미 꽤 고정된 것

### 보스/전투 구조
- 결전 / 극상위 보스 철학
- 공용 패턴 / 전용 패턴 방향
- 데스카운트 구조
- 결과창 구조
- 딜 기여도 집계 방향

### 운영/DB 구조
- 운영 철학
- 로그 / 집계 / 설정 테이블 분리
- API / DTO / 패키지 구조
- 운영 대시보드 MVP 방향

### 생활 구조
- 생활은 보조 성장 루트
- 필드 채집 + 영지 생산 병행
- 청사진 영구 해금
- 요리 / 연금 / 제련 / 영지 생산 방향
- 음식 / 만찬 / 물약 역할 분리

---

## 2. 아직 정확 수치 확정이 필요한 것

### 보스 쪽
- 보스별 체력
- 보스별 방어력
- 패턴 피해 배수
- 패턴 쿨타임
- 무력화 수치
- 발악 타이머
- 보스별 실제 패턴 타임라인

### 생활 쪽
- 자원별 정확 드랍량
- 희귀 부산물 확률
- 영지 시설별 생산량
- 업그레이드 비용
- 제작식 재료 개수
- 숙련도 경험치 테이블
- 거래 기준가

### 음식 / 만찬 / 물약
- 최종 효과 수치
- 재사용 대기시간 세부값
- 재료 소모량
- 제작 난이도
- 등급별 차이 최종값

---

## 3. 추가로 더 세부화하면 좋은 것

### 보스 쪽
- 보스별 패턴 스크립트 표
- 페이즈 전환 조건 상세
- 패턴 우선순위 표
- 보상 드랍 수치표

### 생활 쪽
- 테마별 자원 세부 목록 확정
- 생활 UI 상세 와이어프레임
- 요리/연금 레시피 전체표
- 영지 시설 전체 목록 확정

### 운영 쪽
- 경고 임계값 표
- 대시보드 그래프 우선순위 확정
- 디스코드 봇 명령 최종 목록
- 관리자 권한 단계 확정

---

## 4. Codex에 지금 시켜도 되는 것

- 멀티모듈 프로젝트 뼈대
- 공용 enum / DTO
- DB schema 1차
- admin-api 최소 엔드포인트
- 보스 엔진 공용 구조
- 생활 시스템 공용 구조
- 채집/제작/영지 생산 뼈대
- 결과창 / 데카 UI 뼈대

즉:
**구조 구현은 지금도 충분히 가능하다.**

---

## 5. Codex에 바로 시키지 않는 게 좋은 것

- 최종 밸런스 수치 박기
- 드랍률 완전 확정
- 모든 보스 세부 타임라인 하드코딩
- 생활 최종 가격표 확정
- 레시피/자원 수량 완전 확정

즉:
**최종 수치 확정은 알파/베타 테스트 전후로 잡는 게 더 안전하다.**

---

## 6. 지금 이후 추천 확정 순서

### 1순위
보스별 실제 패턴 스크립트 표

### 2순위
보스 보상 수치표

### 3순위
생활 제작식 전체표

### 4순위
생활 자원 / 시설 최종 목록표

### 5순위
운영 경고 임계값 표

---

## 7. 한 줄 요약

**지금은 구조와 중간 설계가 많이 완료된 상태이고, 앞으로는 보스 패턴 스크립트표·보상 수치표·생활 제작식표처럼 실제 밸런스와 구현을 직접 좌우하는 세부 데이터 표를 순서대로 확정하는 단계로 넘어가면 된다.**
