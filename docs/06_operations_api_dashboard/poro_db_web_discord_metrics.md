# 포로 서버 DB / 웹 / 디스코드 노출 항목 정리 초안

## 문서 목적
이 문서는 포로 서버 운영 철학인 **“운영자가 통제 불가능한 영역에서 무언가 터지면 안 된다”**를 기준으로,
어떤 데이터를 DB에 저장하고, 웹/디스코드/관리자 화면에서 무엇을 확인 가능하게 해야 하는지 정리한 초안이다.

핵심 목표는 다음과 같다.

- 서버 내 핵심 수치를 **운영자가 직접 본다**
- 이상 징후를 **사후가 아니라 조기 감지**한다
- 밸런스 조정이 감이 아니라 **실제 수치 기반**으로 이루어지게 한다
- 웹 / 디스코드 / 관리자 명령이 **같은 원본 데이터**를 보게 한다

---

## 1. 운영 철학

### 1-1. 통제 불가능한 영역을 만들지 않는다
운영자가 모르는 상태에서 아래 같은 일이 벌어지면 안 된다.

- 골드가 비정상적으로 풀림
- 특정 보스가 의도보다 너무 쉽게 깨짐
- 특정 보스가 아예 못 깨질 정도로 과함
- 특정 직업/무기만 비정상적으로 강함
- 큐브/강화석이 지나치게 누적되거나 고갈됨
- 특정 재료만 공급이 막히거나 넘침
- 서버 내 경제가 이미 망가졌는데 늦게 알아챔

즉 핵심은:

**“게임 안에서 일어나는 중요한 변화는 운영자가 수치로 직접 볼 수 있어야 한다.”**

---

### 1-2. 데이터는 보기 위한 저장이어야 한다
DB에 저장만 하고 안 보면 의미가 없다.

그래서 각 데이터는 최소한 아래 셋 중 하나는 만족해야 한다.

- 운영자가 웹에서 바로 볼 수 있음
- 디스코드 봇으로 요약 조회 가능
- 관리자 명령으로 즉시 확인 가능

---

### 1-3. 원본 데이터는 하나로 통일한다
웹, 디스코드, 관리자 툴이 서로 다른 계산값을 보면 안 된다.

반드시:
- DB 원본
- 웹 대시보드
- 디스코드 봇
- 관리자 명령
이 **같은 집계 기준**을 보게 해야 한다.

---

## 2. 노출 우선순위

운영자가 꼭 봐야 하는 항목을 우선순위로 나누면 아래와 같다.

### 1순위: 경제 / 성장 / 보스 핵심 통계
- 서버 전체 골드 총량
- 일일 골드 생성량 / 소멸량
- 강화석 총량 / 일일 증감
- 큐브 총량 / 일일 증감
- 보스별 클리어 타임
- 보스별 클리어율
- 보스별 실패 원인
- 직업/무기별 평균 딜량

### 2순위: 장비 / 강화 / 잠재 / 재료 분포
- 강화 단계 분포
- 잠재 등급 분포
- 장비 티어 분포
- 세트 제작 수량
- 재료별 공급량 / 소비량

### 3순위: 상위 운영 / 시즌 관리 / 기록
- 테마별 결전 / 극상위 최초 클리어
- 명예의 전당
- 시즌별 보스 통계
- 유저 상위권 성장 분포
- 비정상 수치 감지 로그

---

## 3. 꼭 저장해야 하는 DB 항목

## 3-1. 경제 데이터

### 저장 대상
- 유저별 현재 골드
- 서버 전체 골드 총량
- 골드 획득 로그
- 골드 소비 로그
- 거래소/유저 간 거래 로그
- NPC/제작/강화/큐브 사용처별 골드 소모 로그

### 왜 필요한가
운영자가 아래를 바로 봐야 하기 때문이다.

- 서버에 골드가 너무 많이 풀리는지
- 골드가 어디서 많이 생기는지
- 골드가 어디서 가장 많이 소모되는지
- 특정 업데이트 뒤 골드 인플레가 시작됐는지

### 꼭 봐야 하는 화면
- 오늘 골드 생성량
- 오늘 골드 소멸량
- 순증/순감
- 최근 7일 추이
- 생성처 TOP 10
- 소모처 TOP 10

---

## 3-2. 강화 / 큐브 / 잠재 데이터

### 저장 대상
- 유저별 장비 강화 단계
- 강화 시도 횟수
- 강화 성공 횟수
- 단계별 성공률 실측
- 큐브 사용 횟수
- 승급 횟수
- 잠재 등급 분포
- 잠재 옵션 분포
- 잠재 이탈 발생 횟수

### 왜 필요한가
설계 확률과 실제 체감 확률 차이를 운영자가 봐야 한다.

예:
- 18강 도달자가 너무 적은가
- 큐브가 설계보다 너무 빨리 유니크/레전더리를 만드는가
- 특정 옵션이 너무 자주/너무 적게 뜨는가

### 꼭 봐야 하는 화면
- 강화 단계 분포 그래프
- 일일 강화 시도 수
- 단계별 실제 성공률
- 큐브 사용량
- 잠재 등급 분포
- 상위 옵션 출현 비율

---

## 3-3. 장비 / 세트 / 제작 데이터

### 저장 대상
- 유저별 장비 티어(T1/T2)
- 부위별 장착 장비
- 세트 효과 활성화 상태
- 세트 제작 횟수
- 유니크 제작 발생 횟수
- 전승 사용 횟수
- 제작 재료 사용량

### 왜 필요한가
운영자가 실제 성장 속도를 파악해야 하기 때문이다.

예:
- T2 진입이 너무 빠른가
- 특정 세트만 몰리는가
- 유니크 제작 확률 체감이 너무 낮은가
- 특정 제작 재료가 병목인가

### 꼭 봐야 하는 화면
- T1/T2 유저 비율
- 세트별 제작 수량
- 세트별 장착 수량
- 유니크 제작 누적 횟수
- 재료 소비량 / 남은 공급량

---

## 3-4. 보스 데이터

### 저장 대상
- 보스별 입장 횟수
- 보스별 클리어 횟수
- 보스별 실패 횟수
- 보스별 평균 클리어 타임
- 보스별 최소 / 최대 클리어 타임
- 솔플 / 2인 / 3인 클리어율
- 전멸 패턴
- 발악 패턴 진입 비율
- 발악 패턴 성공률
- 보스별 평균 사망 횟수

### 왜 필요한가
보스 난이도는 체감이 아니라 수치로 봐야 한다.

예:
- 어떤 보스가 지나치게 못 깨지는가
- 특정 패턴이 과도하게 사람을 죽이는가
- 1인/3인 보정이 이상한가
- 발악 패턴이 너무 가혹한가

### 꼭 봐야 하는 화면
- 보스별 클리어율
- 보스별 평균 클리어 시간
- 보스별 가장 위험한 패턴 TOP 5
- 파티 인원수별 성공률
- 결전/극상위 최초 클리어 기록

---

## 3-5. 전투 / 딜량 데이터

### 저장 대상
- 직업별 평균 딜량
- 무기별 평균 딜량
- 보스전 기준 총 누적 피해량
- 핵심기 사용 횟수
- 평균 사이클 시간
- 사망 직전 받은 피해 패턴
- 상태이상 적용량
- 무력화 성공 기여량

### 왜 필요한가
운영자가 직업/무기 밸런스를 직접 봐야 하기 때문이다.

예:
- 태도가 설계보다 지나치게 강한가
- 대검이 보스전에서 유독 약한가
- 특정 무기만 발악 패턴 성공률이 높은가
- 암살자/마법사/전사의 격차가 심한가

### 꼭 봐야 하는 화면
- 직업별 평균 딜량
- 무기별 평균 딜량
- 상위/중위/하위 분포
- 보스별 직업 편차
- 평균 사이클 시간

---

## 3-6. 재료 / 인벤토리 / 수급 데이터

### 저장 대상
- 재료별 총 생성량
- 재료별 총 소비량
- 재료별 현재 유통량
- 테마별 재료 공급량
- 결전 보스 재료 공급량
- 극상위 상징 재화 획득량
- 밀수코인 / 마력의 정수 같은 특수 재화 로그

### 왜 필요한가
특정 재료 병목이나 과잉 공급을 빨리 잡아야 한다.

### 꼭 봐야 하는 화면
- 재료별 일일 순증/순감
- 부족한 재료 TOP 10
- 과잉 재료 TOP 10
- 테마별 핵심 재료 공급 현황

---

## 4. 웹 대시보드에서 꼭 보여야 하는 것

## 4-1. 운영 메인 대시보드
이건 로그인 후 가장 먼저 보여야 하는 화면이다.

### 추천 카드
- 현재 서버 전체 골드
- 오늘 골드 순증감
- 오늘 큐브 사용량
- 오늘 강화 시도 수
- 오늘 결전 보스 클리어 수
- 오늘 극상위 도전 수
- 오늘 최초 클리어 여부
- 이상 징후 경고

### 이상 징후 예시
- 골드 순증이 최근 평균 대비 2배 이상
- 특정 보스 클리어율 급락
- 특정 직업 평균 딜량 급상승
- 특정 재료 유통량 급감

---

## 4-2. 경제 대시보드
### 보여야 하는 것
- 총 골드
- 생성/소멸 추이
- 생성처 비율
- 소모처 비율
- 상위 유저 보유량 분포
- 거래량

---

## 4-3. 성장 대시보드
### 보여야 하는 것
- T1/T2 유저 비율
- 강화 단계 분포
- 잠재 등급 분포
- 세트 제작 수량
- 유니크 제작 발생 수

---

## 4-4. 보스 대시보드
### 보여야 하는 것
- 보스별 도전 횟수
- 클리어 횟수
- 클리어율
- 평균 클리어 타임
- 발악 패턴 진입률
- 발악 성공률
- 실패 원인 TOP 5
- 1/2/3인 보정별 성공률

---

## 4-5. 전투 밸런스 대시보드
### 보여야 하는 것
- 직업별 평균 DPS
- 무기별 평균 DPS
- 직업별 보스 클리어 기여도
- 직업별 생존률
- 직업별 발악 성공 기여도

---

## 5. 디스코드 봇에서 바로 볼 수 있어야 하는 것

디스코드는 **빠른 확인용**으로 가야 한다.
웹이 상세 조회용이면, 디코는 요약 조회용이다.

## 5-1. 운영 요약 명령
### 예시
- `/운영 요약`
- `/경제 요약`
- `/보스 요약`
- `/직업 통계`
- `/극상위 현황`

### 예시 출력
- 오늘 골드 순증: +1,240,000
- 결전 클리어 수: 84
- 극상위 도전 수: 12
- 최초 클리어 없음
- 가장 많이 실패한 보스: 엘드하임
- 평균 딜량 1위 무기: 태도

---

## 5-2. 보스 명령
### 예시
- `/보스 통계 모르바인`
- `/보스 통계 라그네스`
- `/보스 통계 카르멘`

### 보여야 하는 것
- 최근 7일 도전 수
- 클리어율
- 평균 클리어 타임
- 가장 위험한 패턴
- 발악 진입률
- 1/2/3인 성공률

---

## 5-3. 경제 명령
### 예시
- `/골드 현황`
- `/재화 현황`
- `/재료 현황`

### 보여야 하는 것
- 총량
- 일일 증감
- 생성처/소모처 상위
- 이상 징후 여부

---

## 5-4. 기록 명령
### 예시
- `/최초 클리어`
- `/명예의 전당`
- `/극상위 진행도`

### 보여야 하는 것
- 결전/극상위 최초 클리어 기록
- 시즌 최고 기록
- 아직 미클리어인 극상위 테마

---

## 6. 관리자 명령으로 즉시 확인할 수 있어야 하는 것

웹/디코 말고, 게임 안에서도 운영자는 봐야 한다.

### 추천 명령
- `/admin eco summary`
- `/admin boss summary <boss_id>`
- `/admin gold flow`
- `/admin class dps`
- `/admin material flow`
- `/admin extreme status`

### 예시 출력
- 서버 전체 골드
- 오늘 생성/소멸
- 특정 보스 평균 클리어 시간
- 직업별 평균 딜량
- 부족한 재료 상위 5개
- 아직 미클리어인 극상위 목록

---

## 7. 이상 징후 감지 항목

이건 꼭 자동 경고가 있어야 한다.

### 경제
- 골드 순증 급증
- 특정 재료 비정상 과잉/고갈
- 특정 거래량 급증

### 성장
- 강화 18강 도달 비율 급락/급등
- 레전더리 잠재 비율 급변
- 특정 세트 제작 비율 과몰림

### 보스
- 특정 보스 클리어율 급락
- 특정 보스 발악 성공률 0% 지속
- 특정 패턴 사망률 비정상 상승

### 전투
- 특정 직업 평균 딜량 급등
- 특정 무기 채용률 비정상 몰림
- 특정 클래스 생존률 급락

---

## 8. 명예 / 기록 시스템 노출

극상위는 업적형 콘텐츠이므로 기록 노출이 중요하다.

### 저장 추천 항목
- 최초 클리어 날짜
- 최초 클리어 파티 / 유저
- 시즌 내 첫 클리어
- 최소 클리어 타임
- 무사망 클리어 여부
- 솔플 클리어 여부
- 발악 패턴 성공 클리어 여부

### 노출 추천 위치
- 웹 명예의 전당
- 디스코드 기록 채널
- 게임 내 기념 NPC
- 보스 입장 전 현황판

## 9. 운영자가 반드시 매일 볼 항목

추천 데일리 체크리스트다.

### 매일 확인
- 골드 순증감
- 결전 클리어 수
- 극상위 도전 수
- 보스별 클리어율 변화
- 직업별 평균 딜량
- 부족 재료/과잉 재료

### 주간 확인
- 강화 단계 분포
- 잠재 등급 분포
- 세트 제작 비율
- 극상위 진행도
- 직업/무기 밸런스 추이

## 10. 한 줄 요약

### 운영 철학
**운영자가 직접 보고 통제할 수 없는 핵심 수치는 존재하면 안 된다.**

### 따라서
- DB에는 원본 로그가 있어야 하고
- 웹에는 상세 대시보드가 있어야 하며
- 디스코드에는 빠른 요약 조회가 있어야 하고
- 게임 안 관리자 명령으로 즉시 확인 가능해야 한다
