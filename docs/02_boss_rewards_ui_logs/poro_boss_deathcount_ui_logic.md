# 포로 서버 보스 데스카운트 / 부활 UI 및 로직 구조 초안

## 문서 목적
이 문서는 포로 서버 보스전에서 사용하는 **데스카운트, 부활, 실패 처리, UI 표시 구조**를 실제 구현 가능한 수준으로 정리한 초안이다.

핵심 목표:
- 이미 정한 데스카운트 규칙을 UI와 로직으로 구체화한다.
- 유저가 지금 남은 데카와 부활 가능 여부를 명확히 알 수 있게 한다.
- 결전/극상위 보스전에 공통으로 재사용 가능한 구조를 만든다.
- Codex가 보스룸 / 전투 엔진 / UI 서비스에 바로 반영할 수 있게 한다.

---

## 1. 확정 규칙 요약

### 1-1. 파티 공용 데스카운트
- 1인: **3 데카**
- 2인: **4 데카**
- 3인: **5 데카**

### 1-2. 사망 시 처리
- 사망 시 공용 데스카운트 1 소모
- 데카가 남아 있으면 부활 가능
- 데카가 0이면 더 이상 부활 불가

### 1-3. 실패 조건
- 데카가 0인 상태에서
- 파티 전원이 사망하면
- 보스전 실패 처리

즉:
- **데카 > 0**: 복귀 가능
- **데카 = 0, 생존자 있음**: 계속 진행
- **데카 = 0, 전원 사망**: 종료

---

## 2. 설계 철학

### 2-1. 유저가 반드시 알아야 하는 정보
보스전에서 아래 정보는 숨기면 안 된다.

- 남은 데스카운트
- 내가 부활 가능한지
- 파티에 생존자가 있는지
- 전멸 직전인지
- 지금 실패한 이유가 데카 소진 때문인지, 전원 사망 때문인지

이 정보가 안 보이면 억울함이 커진다.

---

### 2-2. 데카 UI는 “항상 보이는 정보”여야 한다
체력처럼 데카도 전투 내내 중요한 정보다.

그래서 추천은:
- 보스전 중 항상 보이는 보조 UI
- 사망 시 즉시 뜨는 안내 UI
- 데카 0일 때 강한 경고 UI

---

## 3. 추천 UI 채널

## 3-1. 사이드 상태창 / 보조 HUD
가장 추천하는 상시 표시 방식이다.

### 표시 예시
- `데스카운트: 5`
- `생존 인원: 3/3`
- `부활 가능: 가능`
- `전투 상태: 진행 중`

### 장점
- 항상 확인 가능
- 보스바를 방해하지 않음
- 남은 데카를 계속 인지하게 해줌

---

## 3-2. 액션바
즉시 경고용으로 좋다.

### 사용 예시
- `공용 데스카운트 1 소모`
- `남은 데스카운트: 2`
- `더 이상 부활할 수 없습니다`
- `생존자가 남아 있어 전투는 계속됩니다`

### 용도
- 사망 직후
- 부활 불가 직후
- 전멸 직전

---

## 3-3. 타이틀 / 서브타이틀
중대한 순간에만 사용

### 사용 예시
- 타이틀: `데스카운트 소진`
- 서브타이틀: `이제 더 이상 부활할 수 없습니다`

- 타이틀: `전멸`
- 서브타이틀: `파티 공용 데스카운트가 모두 소진되었습니다`

### 용도
- 데카 0 도달
- 전투 실패
- 마지막 생존자 상황

---

## 3-4. 채팅 로그
기록과 복기용

### 예시
- `[보스전] 파티 공용 데스카운트가 1 감소했습니다. (남은 데카: 3)`
- `[보스전] 현도 님이 쓰러졌습니다.`
- `[보스전] 남은 데스카운트가 없어 더 이상 부활할 수 없습니다.`
- `[보스전] 파티 전원이 전투 불능 상태가 되어 전투에 실패했습니다.`

채팅 로그는 너무 많으면 시끄러우니,
중요 이벤트만 남기는 쪽이 좋다.

---

## 4. 전투 중 UI 흐름

## 4-1. 전투 시작 시
보스전 시작과 동시에 현재 데카 정보를 보여준다.

### 추천 표시
- 액션바: `공용 데스카운트: 5`
- 사이드 UI: `데스카운트 5 / 생존 3/3`

### 목적
시작부터 룰을 분명하게 인식시킴

---

## 4-2. 사망 시
플레이어가 사망하면 즉시 아래 흐름으로 간다.

### 사망 처리 UI
1. 액션바: `공용 데스카운트 1 소모`
2. 액션바 또는 서브타이틀: `남은 데스카운트: N`
3. 죽은 플레이어에게:
   - `부활 가능` 또는 `부활 불가` 안내
4. 파티 전체에:
   - `현도 님이 쓰러졌습니다`

---

## 4-3. 데카 0 도달 시
이 시점은 반드시 크게 보여야 한다.

### 추천 표시
- 타이틀: `데스카운트 소진`
- 서브타이틀: `이제 더 이상 부활할 수 없습니다`
- 액션바: `전원 생존 유지 필요`

### 목적
전투의 국면이 바뀌었음을 명확히 전달

---

## 4-4. 데카 0인데 생존자가 남아 있을 때
이건 포로 서버에서 긴장감이 가장 높은 구간이다.

### 추천 표시
- 액션바: `부활 불가 상태 - 생존자가 전투를 이어갑니다`
- 사이드 UI:
  - `데스카운트: 0`
  - `생존 인원: 1/3`
  - `부활 가능: 불가`

---

## 4-5. 전멸 시
### 추천 표시
- 타이틀: `전멸`
- 서브타이틀: `데스카운트가 모두 소진된 상태에서 전원 사망했습니다`
- 채팅 로그: 실패 원인 출력
- 전투 결과창 또는 보스방 종료 처리

---

## 5. 플레이어 사망 후 상태 구조

## 5-1. 사망 플레이어 상태 분기
플레이어가 죽으면 두 가지다.

### A. 데카가 남아 있음
- 부활 대기 상태
- 일정 시간 후 또는 지정 위치에서 복귀 가능

### B. 데카가 없음
- 관전 상태 또는 전투 불능 상태
- 전투 종료 전까지 복귀 불가

---

## 5-2. 추천 복귀 방식
포로 서버에는 아래 중 하나가 적합하다.

### 추천안
- **짧은 대기 후 보스룸 외곽/안전지점 복귀**
- 또는
- **파티 공용 부활 처리**

현재 구조상 가장 무난한 건
**짧은 대기 후 안전지점 복귀**다.

### 이유
- 너무 즉시 복귀면 죽음 부담이 약해짐
- 너무 오래 대기면 흐름이 끊김

### 권장 대기시간
- **5~8초 추천**
- 1차 추천: **6초**

---

## 6. 로직 구조 추천

## 6-1. 핵심 상태값
보스런 단위로 최소 아래 값이 있어야 한다.

### run context에 필요한 값
- `partySize`
- `maxDeathCount`
- `remainingDeathCount`
- `aliveCount`
- `isBattleFailed`
- `deadPlayerIds`
- `revivablePlayerIds`
- `lockedDeadPlayerIds`  // 데카 0 이후 부활 불가 상태

---

## 6-2. 추천 클래스 구조
### 공용 전투 엔진 쪽
- `BossDeathCountService`
- `BossReviveService`
- `BossBattleFailService`
- `BossDeathUiService`

### 역할
#### `BossDeathCountService`
- 인원수별 데카 초기화
- 사망 시 차감
- 남은 데카 조회

#### `BossReviveService`
- 부활 가능 여부 판단
- 복귀 위치/시간 관리
- 데카 0 시 복귀 차단

#### `BossBattleFailService`
- 전원 사망 여부 판단
- 실패 조건 검사
- 보스전 종료 트리거

#### `BossDeathUiService`
- 액션바/타이틀/상태창 갱신
- 파티원 사망 알림
- 데카 소진 알림

---

## 6-3. 추천 처리 흐름
플레이어 사망 시:

1. 플레이어 사망 이벤트 감지
2. `BossDeathCountService.consumeDeathCount()`
3. `remainingDeathCount` 갱신
4. `aliveCount` 갱신
5. 데카 남았는지 확인
6. 남았으면 revive queue 등록
7. 데카 없으면 locked dead 상태 등록
8. `BossDeathUiService`로 UI 갱신
9. 전원 사망 + 데카 0인지 검사
10. 조건 만족 시 `BossBattleFailService.failBattle()`

---

## 7. UI 데이터 구조 추천

## 7-1. 전투 HUD용 DTO 예시
```java
public record BossDeathCountUiState(
    int maxDeathCount,
    int remainingDeathCount,
    int aliveCount,
    int totalPartyCount,
    boolean reviveAvailable,
    boolean dangerState
) {}
```

### 의미
- `reviveAvailable`: 현재 죽은 플레이어 기준 복귀 가능 여부
- `dangerState`: 데카 0 또는 생존 인원 1인 상태 같은 경고 단계

---

## 7-2. 플레이어 상태 enum 추천
```java
public enum BattleLifeState {
    ALIVE,
    DEAD_REVIVABLE,
    DEAD_LOCKED,
    SPECTATING
}
```

이렇게 두면 관리가 쉽다.

## 8. 사망/복귀 위치 구조

## 8-1. 추천 위치
- 보스룸 중앙 복귀는 비추천
- **보스룸 외곽 안전지점** 추천
- 또는 **전용 복귀 발판**

### 이유
- 즉시 재합류 가능
- 억까 줄임
- 복귀 직후 바로 광역기 맞는 상황 완화

---

## 8-2. 복귀 후 보호 시간 넣을지
추천은 **짧게 넣기**다.

### 권장
- 복귀 후 **1~2초 무적 또는 피해 감소**
- 1차 추천: **1.5초 내외**

### 이유
- 부활 직후 광역기에 바로 죽는 억까 방지
- 하지만 너무 길면 죽음 리스크 감소

---

## 9. 전투 결과창 / 실패 이유 표시

보스전 실패 후 유저는 왜 졌는지 알아야 한다.

### 추천 표시 항목
- 실패 이유: `데스카운트 소진 후 전원 사망`
- 총 사망 횟수
- 남은 보스 체력 %
- 도달 페이즈
- 전투 시간

이건 운영 로그와도 잘 연결된다.

## 10. 운영자가 봐야 하는 로그 포인트

### 저장 추천
- run_id
- player_id
- death_time
- remaining_death_count_after
- was_revivable
- revive_success
- revive_delay_seconds
- final_fail_reason

### 운영에서 볼 수 있는 것
- 어떤 보스가 사망을 많이 유발하는지
- 데카 소진 후 클리어 사례가 있는지
- 복귀 후 즉사 비율이 높은지

---

## 11. 1차 구현 추천 범위

### 꼭 구현
- 공용 데카 초기화
- 사망 시 차감
- 데카 남으면 복귀
- 데카 0이면 복귀 불가
- 데카 0 + 전원 사망 시 실패
- 상시 UI 표시
- 사망/소진/전멸 알림

### 나중에 추가
- 관전자 전용 UI
- 복귀 카메라 연출
- 부활 이펙트
- 상세 전투 결과창 고도화

---

## 12. 한 줄 요약

**포로 서버의 보스 데스카운트 시스템은 파티 공용 데카를 상시 UI로 보여주고, 사망 시 즉시 차감·부활 가능 여부를 명확히 알리며, 데카가 0인 상태에서 전원 사망하면 실패하도록 하는 구조가 가장 적합하다. UI는 사이드 상태창으로 상시 표시하고, 액션바와 타이틀은 사망·데카 소진·전멸 같은 중대 순간에만 강하게 사용하는 것이 좋다.**
