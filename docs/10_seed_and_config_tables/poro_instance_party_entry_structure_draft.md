# 포로 서버 보스방 인스턴스 / 파티 입장 구조 초안

## 문서 목적
이 문서는 자동 매칭 없는 파티 기반 보스 입장 구조를 정리한 초안이다.

핵심 목표:
- 원하는 파티가 자유롭게 결전/극상위에 도전하게 한다.
- 하드한 입장 제한 없이도 안정적인 인스턴스 관리를 한다.
- run record, 결과창, 기록 저장과 연결한다.

---

## 1. 기본 철학

### 1-1. 자동 매칭보다 파티 입장 중심
포로 서버는 MMO 매칭보다
미리 파티를 짜고 입장하는 구조가 더 어울린다.

### 1-2. 입장 제한 최소화
- 파티 조건만 맞으면 입장
- unlock quest만 맞으면 입장
- 주간 횟수 제한 없음
- 실패해도 재도전 가능

---

## 2. 추천 입장 흐름

1. 파티 리더가 입장 NPC와 상호작용
2. 파티원 상태 확인
3. unlock / party size / 현재 전투 여부 확인
4. 새 run_id 생성
5. 인스턴스 맵/공간 생성
6. 파티 전원 이동
7. 보스전 시작

---

## 3. 추천 필드

### party entry session
- `entry_session_id`
- `boss_id`
- `leader_user_id`
- `party_member_ids`
- `created_at`

### boss instance
- `instance_id`
- `run_id`
- `boss_id`
- `party_size`
- `status`
- `started_at`
- `ended_at`

---

## 4. 한 줄 요약

**보스방은 자동 매칭 없이 파티 리더가 입장 NPC를 통해 생성하는 인스턴스 구조로 두고, unlock 확인과 현재 상태 검사만 통과하면 횟수 제한 없이 자유롭게 재도전 가능한 방식이 가장 포로 서버답다.**
